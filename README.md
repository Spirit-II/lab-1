# Задание 1

## Задача 1. Генерация списков

### Текст задачи 

Последовательно вводя числа, сформировать список из их последних цифр.

### Алгоритм решения

Каждое введенное пользователем число делится с остатком на 10, полученное значение заносим в список, который после выводится на экран.

### Тестирование

Скриншоты результата работы программы 

## Задача 2. Рекурсия

### Текст задачи 

Найти минимальную цифру натурального числа.

### Алгоритм решения

Проверяем равно ли число 0, если нет то сравниваем единицы числа с минимальным известным (изначально записанным как "9"). В случае если оно меньше, то записываем как новое минимальное и передаем значение в следующий цикл, поделив изначальное число на 10, отсекая уже проверенную цифру.

# Задание 2

## Задача 1

### Текст задачи

Опишите функции, осуществляющие основные математические операции над комплексными числами (сложение, вычитание, умножение, деление, возведение в степень). Использовать готовый класс Complex запрещено.

### Алгоритм решения

Эти функции реализуют основные математические операции над комплексными числами, представленными как пары (действительная_часть, мнимая_часть):

add - сложение комплексный чисел посредством базового операнда "+", делая отдельно с действительными и мнимыми.

sub - аналогично сложению , но используя "-".

mul - Умножает два комплексных числа по правилу: (a1+b1i)⋅(a2+b2i)=(a1a2−b1b2)+(a1b2+a2b1)i.

div - деление: (da1a2+b1b2)/d​, (b1a2−a1b2​)/d, где знаменатель d = a2² + b2² - это квадрат модуля второго числа.

pow - степень, работает на три случая: 
1) Базовый случай - любое число в степени 0 равно (1.0, 0.0).
2) Рекурсивный случай - умножает число само на себя n раз.
3) Отрицательная степень - выводит текст о не состоянии работать с отрицательными степенями.

### Тестирование

Скриншоты результата работы программы
